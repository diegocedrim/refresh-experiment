Na próxima página você encontrará uma lista com 9 classes afetadas por anomalias de código (code smells). Na literatura, essas anomalias são definidas como uma "indicação superficial que usualmente corresponde a um problema mais profundo no sistema" (Fowler et al., 2009). Os code smells apresentados durante esse experimento são indicadores de problemas estruturais críticos em um sistema de software. Assim, mostramos algumas sequências de refatorações que podem ser aplicadas pelos desenvolvedores para ajuda-los a remover tais problemas estruturais.

Durante esse experimento, apresentamos as sequências de refatorações propostas com o intuito de remover os code smells. O seu papel nesse experimento é avaliar tais sugestões de refatoração. Você deve informar se acha que as sequências de refatorações melhoraram a estrutura dos códigos apresentados ou não. Além disso, você deve prover uma justificativa para cada resposta.
